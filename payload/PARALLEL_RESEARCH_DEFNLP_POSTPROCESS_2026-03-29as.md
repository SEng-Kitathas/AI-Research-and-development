# Parallel Research Path — DEFNLP/Post-Processing Synthesis
_Version: 2026-03-29as_

## Phase I — Cleaning / baseline correction
The deepest signal in this zip is not a new architecture. It is a calibration correction:
- LogReg replaces MLP as the correct external floor for ternary plane data.
- Bound-prototype vs estimated-prototype conditions must be separated.
- Several earlier conclusions were using the wrong floor or blurred conditions.

## Phase II — Interrogation findings
The parallel path contributes four load-bearing updates:
1. Prototype binding is the dominant contributor to TQ2 advantage.
2. MAYBE should be redesigned as abstention/OOD logic, not a co-equal competitive family.
3. Text-encoded planes require text-specific transforms or a different encoder path.
4. Project A depends on the prototype/distillation bridge; that bridge is no longer a nice-to-have.

## Phase III — Post-processing / orphan signal rescue
High-value facts that would be easy to miss:
- Transform family alone underperforms LogReg.
- Prototype gain is larger than transform-family gain.
- The old text paradox was partly a baseline selection error, not just substrate weakness.
- MAYBE raw ablation is directionally informative but operationally wrong.
- The zip does not close the dual-head seam or the K2 multi-path seam; it sharpens them.

## Net doctrine effect
This zip does not overthrow the current base.
It narrows dependencies:
- prototype/reference quality is load-bearing
- MAYBE is uncertainty plumbing
- text is a different regime
- Project A is bridge-dependent
