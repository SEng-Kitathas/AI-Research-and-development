# Weight Substrate Basis
_Version: 2026-03-29c_

## Standing Basis

The project is now defined as **instrumented substrate research** over a canonical TQ2 substrate.

## Canonical substrate

TQ2 remains the canonical substrate because it provides:

- ternary native primitive space
- plane/block object model
- cheap closed operator family
- room for local, global, and lifted interpretation
- compatibility with future spectral and quaternion branches

## Runtime doctrine

Whole-plane, blockwise, and hybrid are no longer treated as mutually exclusive runtime alternatives.

They are now:

- **benchmark isolates** in research mode
- **cooperating inference strata** in runtime mode

## Research doctrine

RB-2 is now structured as:

- RB-2A: whole-plane isolate
- RB-2B: blockwise isolate
- RB-2C: hybrid isolate
- RB-2R: integrated runtime composition

The runtime engine should likely use all three strata in sequence:
blockwise -> whole-plane -> hybrid reconciliation.

## Current best path

1. Freeze TQ2 grammar
2. Implement slow reference semantics for RB-2A/B/C/R
3. Run mixed-task benchmark wave
4. Use results to decide:
   - weighting rules
   - reconciliation policy
   - what Forge must optimize for during translation

## Constraint law

All branches must remain within the cheap closed operator family.
If a branch requires generic heavy arithmetic, it is no longer a TQ2-native branch.

## Practical motto

> Separate to measure.
> Compose to run.
