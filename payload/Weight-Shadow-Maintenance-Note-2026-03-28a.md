# Weight Substrate — Shadow Maintenance Note
_Version: 2026-03-28a_

## SINGLE RULE
If a branch changes, the matrix changes.
If a research seam changes, the crosswalk changes.
If the basis changes, the white paper changes.
If none of those changed, the work probably did not matter.

## SESSION START ORDER
At the start of every meaningful session, load in this order:
1. Basis Document
2. Trace Matrix
3. Research Crosswalk
4. Addendum
5. Maintenance Note
6. White Paper Thesis

## SESSION TYPE DECLARATION
Every session must be declared as one of:
- research
- consolidation
- spec
- build
- audit
- ablation

## WHEN TO UPDATE THE BASIS DOC
Update the basis doc when:
- a substrate invariant changes
- a branch becomes canonical or is demoted
- object model assumptions change
- the evaluation doctrine changes

## WHEN TO UPDATE THE MATRIX
Update the matrix when:
- a new branch is formalized
- a branch changes status
- a requirement or dependency changes
- kill criteria are added or modified
- build pressure changes

## WHEN TO UPDATE THE CROSSWALK
Update the crosswalk when:
- research supports or weakens a seam
- a seam moves from speculative to plausible
- a critique materially changes scope
- an internal genealogy artifact changes how the project is interpreted

## WHEN TO UPDATE THE ADDENDUM
Update the addendum after:
- a major reading pass
- a branch decision
- a major implementation delta
- a meaningful napkin/extraction/genealogy finding

## WHEN TO UPDATE THE WHITE PAPER
Update the white paper when:
- the main thesis changes
- branch framing changes
- external evidence changes the argument
- the project’s current “best case” changes

## BRANCH DISCIPLINE
Every branch should eventually have:
- claim
- object model
- operator family
- packing/encoding
- update rule
- task set
- metrics
- kill criteria

## PROHIBITIONS
Do not:
- collapse branch and substrate into one thing
- call an idea “normative” without promoting it in the matrix
- keep speculative claims in the white paper without crosswalk support
- treat zeta/reversibility claims as settled

## PRESERVATION RULES
- never delete shadow docs; supersede them
- keep old branch names if they are part of the genealogy
- preserve napkin artifacts as evidence inputs
- keep the white paper argumentative, and the matrix factual
