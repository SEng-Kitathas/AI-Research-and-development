# Experiment Pre-Registration Log
_Version: 2026-03-29_

This log must be updated BEFORE data is observed for any confirmatory experiment.
Exploratory experiments use a lighter planning entry (see below).
HARKing (hypothesizing after results are known) is a process violation.

---

## CONFIRMATORY ENTRY TEMPLATE

```
EXPERIMENT ID:           [EXP-YYYYMMDD-NNN]
DATE PRE-REGISTERED:     [timestamp]
MODE:                    CONFIRMATORY
HYPOTHESIS:              [exactly one falsifiable claim — specific and measurable]
WHAT WOULD FALSIFY:      [the kill criterion — specific threshold, not vague]
SUCCESS THRESHOLD:       [minimum result to consider hypothesis supported]
ANALYSIS PLAN:           [exactly how results will be analyzed]
BASELINES:               [internal comparators + external comparator — mandatory]
VARIANTS:                [what changes between conditions]
STRUCTURAL DISTINCTNESS: [explicit confirmation that conditions are genuinely different]
SAMPLE/SEED PLAN:        [N, seeds, splits — pre-stated]
DEVIATION POLICY:        [any deviation must be logged BEFORE analysis]
STATUS:                  REGISTERED / IN PROGRESS / COMPLETE / DEVIATED
```

---

## EXPLORATORY ENTRY TEMPLATE

```
EXPERIMENT ID:           [EXP-YYYYMMDD-NNN]
DATE LOGGED:             [timestamp]
MODE:                    EXPLORATORY
GOAL:                    [what we are looking for — not a prediction]
SCOPE:                   [what corpus, code, or conditions are being examined]
WHAT WOULD BE INTERESTING: [what signals would warrant follow-up]
WHAT WOULD NOT:          [what null result would still be informative]
NOTE:                    Results from this experiment are TENTATIVE.
                         A confirmatory experiment is required before promotion.
STATUS:                  PLANNED / IN PROGRESS / COMPLETE
```
