# TQ2 / CIL Unified Ontology v1
_Version: 2026-03-29w_

## Purpose
Keep terminology unified across lab docs and corpus docs.

## Core substrate terms

### TQ2
Canonical geometric reasoning substrate used in the current lab.

### Exact Plane
Truth-preserving, reversible, provenance-bearing layer.
This is the side that supports residuals, audit, canonicalization, and ledger truth.

### Representational Plane
Compressed, geometric, family-forming, prediction-supporting layer.
This is the side that supports clustering, topology, reuse, and latent reasoning.

### CIL
Cognitive Intersymbolic Ledger.
Native exact memory substrate and persistent ledger core of the architecture.

## Memory terms

### Session Buffer
Transient working capture before consolidation.

### Episode
A durable witnessed event or sequence before semantic promotion.

### Semantic Memory
Promoted, consolidated, family-level memory derived from episodes.

### Archive
Retained superseded or historical memory with provenance intact.

### Consolidation Engine
Center of gravity for memory formation.
Promotes, merges, replays, tests contradictions, and manages supersession.

### Replay Layer
Anti-calcification mechanism used to revisit and restress stored memory.

### Retrieval Compiler
Component that compiles ledger contents into working retrieval packets rather than dumping raw storage into context.

## Reasoning terms

### Latent-Kept Reasoning
Reasoning regime where family/route state remains latent by default and explicit collapse happens only when needed.

### Explicit Collapse
Any discrete surfaced decision such as explicit family decode or routed symbolic output.

### Verification Decode
An explicit collapse triggered by checkpoint, instability, disagreement, or output demand.

### Family State
Current latent evidence state over response families or route families.

### Surfaced Family
The currently explicit family after a decode/collapse event.

### Carry
Cheap recurrent latent state propagated across steps.

### Carry Drift
Magnitude of change in carry state between adjacent steps.

## Refresh / reuse terms

### Refresh
Recompute family evidence from a broader pool.

### Reuse
Operate using a previously retained shortlist or route state rather than full recomputation.

### RegimeCache
Inspiration-only term for stable expensive decision reuse.
Native meaning: short-horizon refresh-vs-reuse scheduling.

### K2 Reuse
Fast two-family shortlist operating band refreshed every 2 steps in the current lab branch.

### K3 Meta Lane
Slower broader shortlist or meta-anchor lane refreshed every 3 steps.

### Offset K2/K3 Schedule
Multirate scheme where K2 handles local fast updates and K3 handles slower broader refresh.

### Fast Lane
The more frequently refreshed shortlist / family-routing lane.

### Meta Lane
The slower broader shortlist / anchor lane that constrains or informs the fast lane.

## Radical / geometry terms

### Radical
Factor or family anchor used in exact and representational memory organization.

### Seed Radical
Initial hand-defined radical.

### Mined Radical
Discovered recurring radical candidate.

### Apex Radical
Promoted radical that repeatedly reduces residuals and survives normalization/contradiction pressure.

### Geometry / Topology
Ordering and neighborhood structure used for memory locality, replay locality, retrieval locality, and family clustering.

## Style rule
Use these terms directly.
Avoid performative synonyms and avoid switching vocabulary without need.
