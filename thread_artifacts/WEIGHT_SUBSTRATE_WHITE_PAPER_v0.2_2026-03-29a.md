# Toward a Discrete Geometric Weight Substrate
_White Paper Thesis v0.2 · 2026-03-29a_

## Abstract
This paper proposes a branch-based research program for a discrete geometric weight substrate in which model expressivity comes primarily from structured composition and cheap closed operators rather than scalar precision. The project assumes neither a single already-proven weight format nor a single training path. Instead, it defines a shared substrate and evaluates multiple research branches over that substrate, including grouped Hamilton-style algebra, plane-block geometric permutation, spectral companion layers, 4D mutagenic lifts, native distillation from standard models, and control-layer coupling.

A major update in this version is the elevation of **weight consumption / native distillation** from a side concern to a first-class architectural bridge. The project now treats ingestion, valuation, locality, and native export as part of the substrate problem rather than as pre-processing trivia.

## 1. Problem
Modern weight fields are usually treated as large scalar arrays optimized through multiply-accumulate heavy arithmetic. This project asks whether a different regime is possible:
- lower bit depth
- more discrete structure
- richer local composition
- cheaper arithmetic
- stronger auditability
- better hardware sympathy
- native ingestion from already-trained external systems

The core question is not whether low-bit weights can exist; that is already supported directionally. The question is whether a structured discrete substrate can achieve useful expressivity through composition and legal transforms while preserving cheap arithmetic, auditability, and an effective capability-ingestion path.

## 2. Core Thesis
> Richness comes from structured discrete composition under cheap closed operators, not from scalar precision.

This yields the following thesis:
1. local weight objects should be structured, not merely scalar
2. legal transforms should remain cheap and closed
3. higher expressivity should arise from combinatory state space and grouped relations
4. higher-order lifts should preserve the same algebraic family
5. external capability must be consumable into native form
6. all concrete formats should be treated as research branches unless proven out

## 3. Shared Substrate
The shared substrate assumes:
- discrete low-bit or ternary-friendly nativity
- structured local objects (blocks, planes, grouped objects)
- cheap closed operations (shift, permutation, add/subtract, clamp, lookup)
- deterministic auditability
- hardware-friendly compact packing and locality
- the ability to act on an object as a whole or on its constituent blocks
- a first-class ingestion bridge from external models/content into native substrate form

## 4. Research Branches
### RB-1 — TQ1 / grouped Hamilton branch
A grouped algebra branch preserving the intuition that weight objects can encode richer inter-component relations than flat scalar arrays.

### RB-2 — TQ2 / plane-block geometric permutation branch
The current build-favored branch. This branch treats blocks and planes as the primitive structured objects and emphasizes permutation, reindexing, block composition, and cheap local transforms.

### RB-3 — Spectral companion branch
Adds a transform-domain or exact-integer spectral lift if it can remain a lift of the same base substrate.

### RB-4 — 4D mutagenic lift
Tests whether the same low-cost local algebra can be lifted into a 4D geometric space for mutation/evolution/search without abandoning closure.

### RB-5 — Native distillation / weight consumption
Consumes standard models or learned content and distills them into the native substrate. This branch is now backed not only by conceptual thread evidence but by a real Forge architecture containing extraction, evaluation, mapping, and meritocratic storage roles.

### RB-6 — Control-layer coupling
Couples the weight substrate to the holon/nanite/salience-routing control layer.

### RB-7 — Scheduler/update ablations
Tests update laws, acceptance criteria, and schedules.

## 5. Genealogy
The project lineage contains at least three convergent streams:

### 5.1 Control lineage
salience → sparsity → stigmergic/mycelial routing → heartbeat → holon/nanite isomorphism

### 5.2 Weight lineage
ternary planes → block decomposition → radical assignment → geometric permutation → spectral intuition → Tesseract / 4D lift

### 5.3 Bridge lineage
consume standard weights → distill content → native low-bit substrate

The early Tesseract MHT thread is important because it records where these lineages start branching in language. It is high-signal evidence, but it is not itself a normative spec.

## 6. The Consumption Bridge
The Forge/consumption seam is no longer a hypothetical bridge. The zip read shows a real architecture containing:
- Surgeon (functional extraction)
- Scientist (quality evaluation)
- Engineer (address/locality mapping)
- Artificer (meritocratic storage)
- a documented seven-stage ingestion pipeline

However, this current Forge line is still mostly pre-TQ2 in its assumptions. It scores by entropy/source/position/type more than by geometric resonance, and it stores native blobs/metadata rather than a TQ2/quaternion-native substrate form.

Therefore the next phase is not “invent Forge.” It is “rewrite Forge for the weight substrate.”

## 7. Current Position
The current position is:
- the substrate is canonical
- the branches are experimental
- TQ2 is the current build-favored branch
- TQ1 remains a baseline
- spectral, 4D, control, and distillation branches stay alive unless falsified
- zeta scheduling and strong software reversibility claims remain bounded and test-driven
- Forge/consumption is a first-class branch and must be rewritten toward native TQ2 export
